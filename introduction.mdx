## What is AVA?

AVA (AI Virtual Assistant) is an enterprise-grade AI assistant that runs securely within your Azure tenant. Unlike traditional SaaS AI tools, AVA integrates directly with your Microsoft 365 environment and business systems, using your existing identity and permissions to provide a truly personalized, agentic experience.

## The AVA Advantage

<AccordionGroup>
  <Accordion title="Runs in Your Azure Tenant">
    AVA is deployed as an Enterprise Application within your own Azure environment. Your data never leaves your tenant, ensuring maximum security and compliance.
  </Accordion>
  <Accordion title="Identity-Based Security">
    Uses Microsoft Entra ID for authentication. AVA only accesses what you can access—it operates with your delegated permissions, not elevated privileges.
  </Accordion>
  <Accordion title="No Separate Agents Required">
    Unlike other AI platforms, AVA doesn't require you to create separate agents for each integration. It seamlessly orchestrates across all your connected data sources in a single conversation.
  </Accordion>
  <Accordion title="Seamless Microsoft 365 Integration">
    Native integration with Outlook, Teams, OneDrive, SharePoint, and the entire Microsoft 365 ecosystem.
  </Accordion>
</AccordionGroup>

## Technical Foundation

AVA is built on modern, enterprise-grade architecture:

- **Azure Container Apps**: Scalable, containerized Python backend
- **PostgreSQL with pgvector**: Efficient vector storage for RAG capabilities
- **Model Context Protocol (MCP)**: Standardized integration layer for all data sources
- **Microsoft Entra ID**: Enterprise identity and access management
- **Azure Key Vault**: Secure secrets management

  Learn more about AVA's architecture in our [deployment guide](/deployment/architecture).
