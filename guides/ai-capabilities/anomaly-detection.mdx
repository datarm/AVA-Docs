## Access Anomaly Detection

Detect unusual access patterns:

```python
from ava import Client

client = Client(api_key="YOUR_API_KEY")

# Configure anomaly detection
client.anomaly_detection.configure(
    enabled=True,
    sensitivity="high",
    categories=[
        "unusual_time",
        "unusual_location",
        "unusual_volume",
        "privilege_escalation"
    ]
)

# Get detected anomalies
anomalies = client.anomaly_detection.get_anomalies(
    days=7,
    severity=["high", "critical"]
)

for anomaly in anomalies:
    print(f"Anomaly: {anomaly.type}")
    print(f"User: {anomaly.user}")
    print(f"Resource: {anomaly.resource}")
    print(f"Risk Score: {anomaly.risk_score}")
```

## Anomaly Types

### Access Pattern Anomalies

- **Unusual Time**: Access outside normal business hours
- **Unusual Location**: Access from unexpected geographic location
- **Bulk Export**: Large data export by user
- **Failed Attempts**: Multiple failed access attempts
- **Privilege Escalation**: Accessing resources above normal permissions

### Data Anomalies

- **Schema Changes**: Unexpected table or column modifications
- **Data Drift**: Changes in data distribution
- **Quality Degradation**: Increase in null values or errors
- **Volume Spikes**: Sudden increases in data volume

## Configuring Detection

```yaml
anomaly_detection:
  enabled: true
  baseline_period: 30 days
  sensitivity: medium  # low, medium, high

  access_anomalies:
    unusual_time:
      enabled: true
      business_hours: "08:00-18:00"
      timezone: "America/New_York"

    unusual_location:
      enabled: true
      allowed_countries: ["US", "CA", "GB"]
      alert_on_vpn: true

    bulk_export:
      enabled: true
      threshold: 10000 rows
      require_justification: true

  data_anomalies:
    schema_changes:
      enabled: true
      alert_on_drop: true
      require_approval: true

    data_quality:
      enabled: true
      null_threshold: 0.1
      alert_on_degradation: true
```

## Automated Response

Configure automated responses to anomalies:

```bash
# Create anomaly response rule
ava anomaly-response create \
  --anomaly-type "bulk_export" \
  --severity "high" \
  --action "require_mfa,notify_security_team" \
  --webhook "https://your-app.com/security-alerts"
```

## Investigation

Investigate detected anomalies:

```bash
# View anomaly details
ava anomaly investigate ANO-12345 --show-timeline

# Get related events
ava anomaly related-events ANO-12345 --days 7

# Mark as false positive
ava anomaly mark-false-positive ANO-12345 --reason "Approved bulk export"

# Create incident
ava anomaly create-incident ANO-12345 \
  --severity critical \
  --assignee security-team@company.com
```
