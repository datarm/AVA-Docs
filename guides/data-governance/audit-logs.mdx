## Overview

AVA maintains detailed audit logs of all activities within the platform, providing complete visibility into who accessed what data, when, and why. These audit trails are essential for security investigations, compliance reporting, and forensic analysis.

## Audit Log Types

### User Activity Logs

Track user authentication and actions:

```bash
# View user activity
ava audit user-activity \
  --user john@company.com \
  --days 30 \
  --include-failed-attempts

# Show login history
ava audit logins \
  --user john@company.com \
  --show-locations \
  --show-devices
```

### Data Access Logs

Monitor all data access and queries:

```bash
# View data access logs
ava audit data-access \
  --source salesforce \
  --days 7 \
  --include-queries

# Show sensitive data access
ava audit sensitive-access \
  --classification "PII,PHI" \
  --users all \
  --alert-on-bulk-export
```

### System Audit Logs

Track system configuration changes:

```bash
# View system changes
ava audit system-changes \
  --type "config,integrations,policies" \
  --days 30

# Show permission changes
ava audit permission-changes \
  --show-before-after
```

### Compliance Audit Logs

Compliance-specific audit trails:

```bash
# GDPR audit trail
ava audit compliance \
  --framework gdpr \
  --events "dsar,consent,deletion" \
  --export gdpr-audit-trail.pdf

# HIPAA audit trail
ava audit compliance \
  --framework hipaa \
  --events "phi-access,breach-investigation" \
  --period 2024-01-01:2024-12-31
```

## Configuring Audit Logging

### Enable Comprehensive Logging

```yaml
audit_configuration:
  enabled: true
  retention_period: 7 years

  log_levels:
    authentication: verbose
    data_access: detailed
    admin_actions: comprehensive
    api_calls: standard
    policy_enforcement: detailed

  capture:
    - event_type
    - timestamp
    - user_identity
    - ip_address
    - user_agent
    - resource_accessed
    - action_performed
    - result_status
    - request_payload
    - response_summary
    - session_id
    - geolocation

  storage:
    primary: postgresql
    archive: s3
    encryption: aes-256
    immutable: true

  alerting:
    failed_logins_threshold: 5
    bulk_export_alert: true
    privilege_escalation_alert: true
    unusual_access_patterns: true
```

Apply configuration:
```bash
ava audit configure --config audit-config.yaml
```

## Searching Audit Logs

### Advanced Search Queries

<Tabs>
  <Tab title="CLI Search">
    ```bash
    # Search by user and timeframe
    ava audit search \
      --user john@company.com \
      --start "2024-10-20" \
      --end "2024-10-23" \
      --event-type "data_access"

    # Search by resource
    ava audit search \
      --resource "salesforce.contacts" \
      --action "export" \
      --days 7

    # Search by IP address
    ava audit search \
      --ip "203.0.113.*" \
      --show-users \
      --show-locations

    # Complex search
    ava audit search \
      --query "event_type:policy_violation AND severity:high" \
      --days 30 \
      --export violations.csv
    ```
  </Tab>

  <Tab title="API Search">
    ```python
    from ava import Client

    client = Client(api_key="YOUR_API_KEY")

    # Search audit logs
    results = client.audit.search(
        filters={
            "event_type": ["data_access", "data_export"],
            "classification": ["PII", "PHI"],
            "severity": "high",
            "date_range": {
                "start": "2024-10-01",
                "end": "2024-10-31"
            }
        },
        sort_by="timestamp",
        sort_order="desc",
        limit=1000
    )

    for event in results:
        print(f"{event.timestamp}: {event.user} accessed {event.resource}")
        if event.risk_score > 70:
            print(f"  WARNING: Risk score {event.risk_score}")
    ```
  </Tab>

  <Tab title="SQL Queries">
    ```sql
    -- Direct SQL access for analysis
    SELECT
      timestamp,
      user_email,
      event_type,
      resource_name,
      action,
      risk_score
    FROM audit_logs
    WHERE
      event_type = 'data_access'
      AND classification @> ARRAY['PII']
      AND timestamp >= NOW() - INTERVAL '30 days'
    ORDER BY risk_score DESC
    LIMIT 100;
    ```
  </Tab>
</Tabs>

## Real-Time Monitoring

### Live Audit Stream

```bash
# Monitor audit logs in real-time
ava audit tail --follow

# Filter real-time logs
ava audit tail \
  --event-type "policy_violation,failed_login" \
  --severity "high,critical" \
  --follow

# Alert on specific patterns
ava audit tail \
  --pattern "bulk_export" \
  --alert-webhook "https://your-app.com/alerts" \
  --follow
```

### Automated Alerts

Configure automated alerts for suspicious activities:

```yaml
audit_alerts:
  - name: Failed Login Attempts
    condition: event_type = 'failed_login' AND count > 5 within 10 minutes
    severity: high
    action:
      - send_email: security-team@company.com
      - create_incident
      - block_ip_temporary

  - name: Bulk Data Export
    condition: action = 'export' AND rows_exported > 10000
    severity: medium
    action:
      - send_notification
      - require_justification
      - alert_data_owner

  - name: Privilege Escalation
    condition: action = 'role_assigned' AND role IN ('admin', 'data-governance-officer')
    severity: high
    action:
      - send_alert: ciso@company.com
      - require_approval
      - audit_review_required

  - name: Unusual Access Time
    condition: access_time NOT BETWEEN '06:00' AND '22:00' AND classification = 'SENSITIVE'
    severity: medium
    action:
      - verify_mfa
      - log_detailed
      - notify_manager
```

## Audit Analytics

### Access Patterns Analysis

```python
# Analyze access patterns
analytics = client.audit.analytics(
    period="30d",
    analysis_types=[
        "access_frequency",
        "unusual_patterns",
        "risk_trends"
    ]
)

# User behavior analysis
user_behavior = client.audit.analyze_user_behavior(
    user="john@company.com",
    baseline_period="90d",
    detect_anomalies=True
)

print(f"Anomalies detected: {len(user_behavior.anomalies)}")
for anomaly in user_behavior.anomalies:
    print(f"  {anomaly.type}: {anomaly.description}")
    print(f"  Risk Score: {anomaly.risk_score}")
```

## Audit Log Retention

### Retention Policies

```bash
# Configure retention
ava audit retention configure \
  --default-retention 7-years \
  --compliance-logs 10-years \
  --archive-after 1-year \
  --archive-location s3://audit-archive

# View retention policy
ava audit retention show

# Export old logs to archive
ava audit archive \
  --older-than 365d \
  --destination s3://audit-archive/2023 \
  --compress \
  --verify
```

### Immutable Audit Logs

Ensure audit logs cannot be tampered with:

```yaml
immutability_config:
  enabled: true
  method: blockchain-anchoring  # or write-once-read-many
  verification: cryptographic-hashing

  blockchain:
    provider: ethereum
    frequency: hourly
    smart_contract: "0x..." # Contract address

  integrity_checks:
    frequency: daily
    alert_on_tampering: true
    auto_lock_on_violation: true
```

## Forensic Investigation

### Incident Investigation Workflow

<Steps>
  <Step title="Identify Incident">
    ```bash
    # Search for suspicious activity
    ava audit search \
      --query "user:suspicious@external.com OR ip:203.0.113.*" \
      --days 90
    ```
  </Step>

  <Step title="Collect Evidence">
    ```bash
    # Create investigation case
    ava investigation create \
      --name "Suspicious Access Investigation" \
      --incident-id "INC-2024-001" \
      --start-date "2024-10-20"

    # Collect relevant logs
    ava investigation collect-logs \
      --case "INC-2024-001" \
      --users "suspicious@external.com" \
      --resources "customer_database" \
      --preserve-chain-of-custody
    ```
  </Step>

  <Step title="Analyze Timeline">
    ```bash
    # Generate timeline
    ava investigation timeline \
      --case "INC-2024-001" \
      --show-related-events \
      --visualize

    # Export for analysis
    ava investigation export \
      --case "INC-2024-001" \
      --format forensic-package \
      --include-hashes
    ```
  </Step>

  <Step title="Generate Report">
    ```bash
    # Create investigation report
    ava investigation report \
      --case "INC-2024-001" \
      --include-recommendations \
      --output Investigation-Report-INC-2024-001.pdf
    ```
  </Step>
</Steps>

    ```bash
      --hec-url "https://splunk.company.com:8088" \
      --hec-token "YOUR_HEC_TOKEN" \
    ```

    ```bash
      --api-key "YOUR_API_KEY" \
      --index-pattern "ava-audit-*" \
      --ilm-policy "ava-audit-retention"
    ```


## Troubleshooting

<AccordionGroup>
  <Accordion title="Missing audit logs">
    ```bash
    # Check logging configuration
    ava audit status

    # Verify log storage
    ava audit storage-status

    # Check for gaps in logs
    ava audit verify-completeness \
      --start "2024-10-01" \
      --end "2024-10-31"
    ```
  </Accordion>

  <Accordion title="High audit log volume">
    ```bash
    # Analyze log volume
    ava audit analyze-volume --days 7

    # Configure sampling for high-volume events
    ava audit configure-sampling \
      --event-type "query_executed" \
      --sample-rate 0.1

    # Archive old logs
    ava audit archive --older-than 90d
    ```
  </Accordion>

  <Accordion title="Slow audit log searches">
    ```bash
    # Check index status
    ava audit index-status

    # Rebuild indexes
    ava audit reindex --async

    # Optimize search
    ava audit optimize-storage
    ```
  </Accordion>
