## Risk Categories

### Data Security Risks

- **Unencrypted Sensitive Data**: PII/PHI without encryption
- **Overly Permissive Access**: Too many users with access
- **Public Exposure**: Data accessible without authentication
- **Weak Authentication**: Resources without MFA
- **Stale Credentials**: Unused service accounts with active access

### Compliance Risks

- **Policy Violations**: Data not meeting governance policies
- **Retention Non-compliance**: Data kept beyond retention period
- **Missing Consents**: PII without proper consent records
- **Unclassified Data**: Sensitive data without classifications
- **Audit Gaps**: Missing audit trails for sensitive access

### Operational Risks

- **Shadow IT**: Unmanaged data sources
- **Data Quality Issues**: Incomplete or inaccurate data
- **Orphaned Data**: Data without clear ownership
- **Broken Lineage**: Unknown data dependencies
- **Performance Bottlenecks**: Systems under stress

## Risk Scoring

AVA calculates risk scores (0-100) based on multiple factors:

```python
from ava import Client

client = Client(api_key="YOUR_API_KEY")

# Get risk score for data asset
asset = client.data_catalog.get("salesforce.contacts")
print(f"Risk Score: {asset.risk_score}")
print(f"Risk Level: {asset.risk_level}")  # low, medium, high, critical

# View risk factors
for factor in asset.risk_factors:
    print(f"  {factor.category}: {factor.description} (impact: {factor.impact})")
```

## Risk Dashboard

View organizational risk posture:

```bash
# Open risk dashboard
ava risk dashboard

# View high-risk assets
ava risk list --level critical,high --limit 50

# Risk by category
ava risk summary --group-by category

# Trending risks
ava risk trends --period 30d
```

## Automated Risk Detection

Configure automated risk detection rules:

```yaml
risk_detection_rules:
  - name: Unencrypted PII Detection
    trigger:
      classification: [PII, PII_EMAIL, PII_SSN]
      encryption: false
    severity: critical
    auto_remediate: enable_encryption
    alert: security-team@company.com

  - name: Excessive Access
    trigger:
      user_count: "> 100"
      classification: SENSITIVE
    severity: high
    action: request_access_review
    notify: data-owner

  - name: Stale Data Detection
    trigger:
      last_accessed: "> 365 days"
      classification: [PII, PHI]
    severity: medium
    action: tag_for_review
    suggest: archive_or_delete
```
