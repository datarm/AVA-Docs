## Overview

Once you've connected your data sources, AVA enables you to query, analyze, and govern your data through multiple interfaces. This guide walks you through executing your first queries using AVA's natural language interface, API, and SDK.

<Info>
  **Prerequisites:** Complete the [Connecting Data Sources](/getting-started/connecting-data-sources) guide before proceeding.
</Info>

## Natural Language Queries

AVA's AI-powered interface allows you to query data using plain English:

### Using the Dashboard

<Steps>
  <Step title="Open Query Interface">
    Navigate to **Data Catalog** > **Query** in your AVA dashboard
  </Step>

  <Step title="Enter Your Question">
    Type a natural language query, such as:
    - "Show me all tables containing customer email addresses"
    - "What PII data exists in our Salesforce instance?"
    - "List databases with HIPAA-regulated data"
    - "Find data sources accessed by the marketing team"
  </Step>

  <Step title="Review Results">
    AVA will:
    - Interpret your query
    - Search across all connected data sources
    - Return relevant results with context
    - Highlight sensitive data classifications
  </Step>

  <Step title="Refine Query">
    Click **Refine** to add filters or adjust the query scope
  </Step>
</Steps>

<Check>
  Your results will include data lineage, access permissions, and risk scores!
</Check>

## API Queries

Execute queries programmatically using AVA's REST API:

### Basic Query

```bash
curl -X POST https://ava.your-domain.com/api/v1/query \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Find all tables with customer email addresses",
    "data_sources": ["salesforce", "snowflake"],
    "filters": {
      "classification": ["PII", "SENSITIVE"],
      "risk_level": ["high", "critical"]
    }
  }'
```

### Response Format

```json
{
  "query_id": "qry_abc123",
  "status": "completed",
  "results": [
    {
      "source": "salesforce",
      "type": "table",
      "name": "Contact",
      "schema": "default",
      "fields": [
        {
          "name": "Email",
          "type": "string",
          "classification": ["PII", "EMAIL"],
          "risk_score": 75
        }
      ],
      "metadata": {
        "row_count": 150000,
        "last_modified": "2024-10-20T15:30:00Z",
        "owner": "salesops@company.com"
      },
      "access": {
        "users": 45,
        "roles": ["sales", "marketing", "admin"]
      }
    }
  ],
  "total_results": 12,
  "execution_time_ms": 234
}
```

### Advanced Query with Filters

```bash
curl -X POST https://ava.your-domain.com/api/v1/query \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": {
      "type": "search",
      "criteria": {
        "classification": ["PII", "PHI"],
        "tags": ["customer", "healthcare"],
        "source_types": ["database", "object_storage"]
      },
      "filters": {
        "date_range": {
          "field": "last_accessed",
          "start": "2024-10-01",
          "end": "2024-10-23"
        },
        "risk_score": {
          "min": 70
        }
      }
    },
    "options": {
      "include_lineage": true,
      "include_access_logs": true,
      "limit": 50,
      "offset": 0
    }
  }'
```

## SDK Queries

Use AVA's SDKs for type-safe queries in your applications:

<Tabs>
  <Tab title="Python">
    ```python
    from ava import Client

    # Initialize client
    client = Client(api_key="YOUR_API_KEY")

    # Simple natural language query
    results = client.query("Find all customer PII in Salesforce")
    for result in results:
        print(f"{result.source}/{result.name}: {result.risk_score}")

    # Structured query with filters
    results = client.data_catalog.search(
        classifications=["PII", "SENSITIVE"],
        sources=["salesforce", "snowflake"],
        risk_level="high",
        include_lineage=True
    )

    # Access query results
    for asset in results:
        print(f"Asset: {asset.full_name}")
        print(f"  Classification: {asset.classification}")
        print(f"  Risk Score: {asset.risk_score}")
        print(f"  Users with Access: {len(asset.access.users)}")

        if asset.lineage:
            print(f"  Upstream Sources: {asset.lineage.upstream}")
            print(f"  Downstream Consumers: {asset.lineage.downstream}")

    # Filter-based query
    results = client.data_catalog.filter(
        lambda asset: asset.risk_score > 80 and
                     "email" in asset.name.lower()
    )
    ```
  </Tab>

  <Tab title="JavaScript">
    ```javascript
    import { AVAClient } from '@datarm/ava-sdk';

    // Initialize client
    const client = new AVAClient({ apiKey: 'YOUR_API_KEY' });

    // Simple natural language query
    const results = await client.query(
      'Find all customer PII in Salesforce'
    );

    results.forEach(result => {
      console.log(`${result.source}/${result.name}: ${result.riskScore}`);
    });

    // Structured query with filters
    const searchResults = await client.dataCatalog.search({
      classifications: ['PII', 'SENSITIVE'],
      sources: ['salesforce', 'snowflake'],
      riskLevel: 'high',
      includeLineage: true
    });

    // Access query results
    for (const asset of searchResults) {
      console.log(`Asset: ${asset.fullName}`);
      console.log(`  Classification: ${asset.classification}`);
      console.log(`  Risk Score: ${asset.riskScore}`);
      console.log(`  Users with Access: ${asset.access.users.length}`);

      if (asset.lineage) {
        console.log(`  Upstream: ${asset.lineage.upstream}`);
        console.log(`  Downstream: ${asset.lineage.downstream}`);
      }
    }

    // Complex filtering
    const highRiskAssets = await client.dataCatalog.filter(
      asset => asset.riskScore > 80 &&
               asset.name.toLowerCase().includes('email')
    );
    ```
  </Tab>

  <Tab title="Java">
    ```java
    import com.datarm.ava.Client;
    import com.datarm.ava.model.*;

    // Initialize client
    Client client = new Client("YOUR_API_KEY");

    // Simple natural language query
    QueryResults results = client.query(
        "Find all customer PII in Salesforce"
    );

    for (DataAsset asset : results.getAssets()) {
        System.out.println(
            asset.getSource() + "/" +
            asset.getName() + ": " +
            asset.getRiskScore()
        );
    }

    // Structured query
    SearchRequest request = SearchRequest.builder()
        .classifications(List.of("PII", "SENSITIVE"))
        .sources(List.of("salesforce", "snowflake"))
        .riskLevel(RiskLevel.HIGH)
        .includeLineage(true)
        .build();

    SearchResults searchResults = client.getDataCatalog().search(request);

    // Process results
    for (DataAsset asset : searchResults.getAssets()) {
        System.out.println("Asset: " + asset.getFullName());
        System.out.println("  Classification: " + asset.getClassification());
        System.out.println("  Risk Score: " + asset.getRiskScore());
        System.out.println("  Users: " + asset.getAccess().getUsers().size());

        if (asset.getLineage() != null) {
            System.out.println("  Upstream: " + asset.getLineage().getUpstream());
            System.out.println("  Downstream: " + asset.getLineage().getDownstream());
        }
    }
    ```
  </Tab>
</Tabs>

## Query Builder Interface

For complex queries, use AVA's visual query builder:

<Steps>
  <Step title="Open Query Builder">
    Click **Advanced Query** in the Query interface
  </Step>

  <Step title="Select Data Sources">
    Choose which sources to query (Salesforce, Snowflake, etc.)
  </Step>

  <Step title="Add Filters">
    Build your query visually:
    - **Classification**: PII, PHI, Financial, etc.
    - **Risk Level**: Low, Medium, High, Critical
    - **Date Range**: Created/Modified/Accessed dates
    - **Owner**: Data owner or team
    - **Tags**: Custom tags
    - **Access**: Users or roles with access
  </Step>

  <Step title="Preview Results">
    Click **Preview** to see a sample of results before running the full query
  </Step>

  <Step title="Execute & Export">
    Run the query and export results to CSV, JSON, or Excel
  </Step>
</Steps>

## Query Types

### Data Discovery

Find data assets across your environment:

```python
# Python example
results = client.discovery.find_sensitive_data(
    classifications=["SSN", "CCN", "EMAIL"],
    min_risk_score=70
)
```

### Access Analysis

Understand who has access to what:

```python
# Find overly permissive access
results = client.access.analyze(
    threshold=100,  # More than 100 users
    include_groups=True
)
```

### Compliance Check

Verify compliance status:

```python
# Check GDPR compliance
compliance = client.compliance.check(
    framework="GDPR",
    scope=["salesforce", "snowflake"]
)
print(f"Compliance Score: {compliance.score}%")
print(f"Issues: {compliance.issues_count}")
```

### Risk Assessment

Identify high-risk data:

```python
# Find critical risks
risks = client.risk.assess(
    severity=["high", "critical"],
    categories=["access", "encryption", "retention"]
)
```

## Save & Share Queries

### Save Queries

```python
# Save a query for reuse
client.queries.save(
    name="High Risk PII Assets",
    query={
        "classifications": ["PII"],
        "risk_level": "high"
    },
    schedule="daily"  # Optional: schedule automatic execution
)
```

### Share Queries

```bash
# Share query with team
ava query share \
  --query-id qry_abc123 \
  --team data-governance \
  --permission read

# Create public query link
ava query create-link \
  --query-id qry_abc123 \
  --expires-in 7d
```

## Query Automation

### Scheduled Queries

Run queries automatically on a schedule:

```python
# Schedule daily query
schedule = client.queries.schedule(
    query_id="qry_abc123",
    schedule="0 9 * * *",  # Daily at 9 AM
    notify=["data-governance@company.com"],
    export_to="s3://reports/daily-risk-assessment/"
)
```

### Webhook Integration

Get notified when query conditions are met:

```python
# Create webhook for high-risk alerts
webhook = client.webhooks.create(
    name="High Risk Alert",
    trigger="query_result",
    query={
        "risk_score": {"min": 90}
    },
    url="https://your-app.com/webhooks/ava-alerts",
    events=["new_high_risk_asset"]
)
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Query returns no results">
    **Check these items:**
    1. Verify data sources are connected and synced
    2. Check filters aren't too restrictive
    3. Ensure you have permissions to view the data
    4. Verify data classifications are up to date
  </Accordion>

  <Accordion title="Query is too slow">
    **Optimization tips:**
    1. Limit query scope to specific sources
    2. Add more specific filters
    3. Use indexed fields in filters
    4. Consider scheduling heavy queries
    ```python
    # Bad: queries all sources
    results = client.query("Find PII")

    # Better: specific scope
    results = client.query(
        "Find PII",
        sources=["salesforce"],
        limit=100
    )
    ```
  </Accordion>

  <Accordion title="Unexpected results">
    **Validation steps:**
    1. Check data classification accuracy
    2. Verify query syntax
    3. Review filters and conditions
    4. Check for stale cached results
    ```bash
    # Clear query cache
    ava query clear-cache --query-id qry_abc123

    # Force refresh
    ava query refresh --query-id qry_abc123
    ```
  </Accordion>
</AccordionGroup>
