## Before You Begin

<Info>
  **Prerequisites:**
  - Admin access to AVA
  - Credentials for your data sources
  - Network access from AVA to your data sources
  - Appropriate permissions on source systems
</Info>

### Network Configuration

Ensure AVA can reach your data sources:

<Tabs>
  <Tab title="Cloud Deployment">
    **Firewall Rules:**
    - Whitelist AVA's IP addresses in your data source firewall
    - Allow outbound connections on required ports

    ```bash
    ava network get-ips

    # Test connectivity
    ava network test --host your-db-host --port 5432
    ```

## Salesforce Integration

Connect to Salesforce for CRM data governance:

<Steps>
  <Step title="Create Connected App">
    In Salesforce Setup:
    1. Navigate to **App Manager**
    2. Click **New Connected App**
    3. Fill in basic information:
       - **Connected App Name**: AVA by DataRM
       - **API Name**: AVA_DataRM
       - **Contact Email**: your-email@company.com
  </Step>

  <Step title="Configure OAuth Settings">
    Enable OAuth:
    - **Enable OAuth Settings**: Checked
    - **Callback URL**: `https://ava.your-domain.com/integrations/salesforce/callback`
    - **Selected OAuth Scopes**:
      - Full access (full)
      - Manage user data via APIs (api)
      - Perform requests at any time (refresh_token, offline_access)
  </Step>

  <Step title="Get Credentials">
    After saving, copy:
    - Consumer Key (Client ID)
    - Consumer Secret (Client Secret)
  </Step>

  <Step title="Connect in AVA">
    ```bash
    ava connect salesforce \
      --name "Production Salesforce" \
      --instance-url "https://your-instance.salesforce.com" \
      --client-id "YOUR_CONSUMER_KEY" \
      --client-secret "YOUR_CONSUMER_SECRET" \
      --environment "production"
    ```

    Or via API:
    ```bash
    curl -X POST https://ava.your-domain.com/api/v1/data-sources \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "type": "salesforce",
        "name": "Production Salesforce",
        "config": {
          "instance_url": "https://your-instance.salesforce.com",
          "client_id": "YOUR_CONSUMER_KEY",
          "client_secret": "YOUR_CONSUMER_SECRET",
          "environment": "production"
        }
      }'
    ```
  </Step>

  <Step title="Initiate Discovery">
    ```bash
    ava scan start --source salesforce --profile full
    ```
  </Step>
</Steps>

<Check>
  AVA will begin discovering and cataloging your Salesforce data!
</Check>

## Snowflake Integration

Connect to Snowflake data warehouse:

<Tabs>
  <Tab title="Using Key Pair Authentication">
    ```bash
    # Generate key pair
    openssl genrsa -out snowflake_key.pem 2048
    openssl rsa -in snowflake_key.pem -pubout -out snowflake_key.pub

    # Upload public key to Snowflake
    # ALTER USER ava_service SET RSA_PUBLIC_KEY='MIIBIjANBg...';

    # Connect AVA
    ava connect snowflake \
      --name "Production Snowflake" \
      --account "your-account.snowflakecomputing.com" \
      --user "ava_service" \
      --private-key-path /path/to/snowflake_key.pem \
      --warehouse "COMPUTE_WH" \
      --database "PROD_DB" \
      --role "AVA_ROLE"
    ```
  </Tab>

  <Tab title="Using Password Authentication">
    ```bash
    ava connect snowflake \
      --name "Production Snowflake" \
      --account "your-account.snowflakecomputing.com" \
      --user "ava_service" \
      --password "YOUR_PASSWORD" \
      --warehouse "COMPUTE_WH" \
      --database "PROD_DB" \
      --role "AVA_ROLE"
    ```
  </Tab>

  <Tab title="Using OAuth">
    ```bash
    ava connect snowflake \
      --name "Production Snowflake" \
      --account "your-account.snowflakecomputing.com" \
      --oauth-client-id "YOUR_CLIENT_ID" \
      --oauth-client-secret "YOUR_CLIENT_SECRET" \
      --oauth-token-url "https://your-account.snowflakecomputing.com/oauth/token-request" \
      --warehouse "COMPUTE_WH"
    ```
  </Tab>
</Tabs>

### Required Snowflake Permissions

```sql
-- Create role for AVA
CREATE ROLE AVA_ROLE;

-- Grant necessary privileges
GRANT USAGE ON WAREHOUSE COMPUTE_WH TO ROLE AVA_ROLE;
GRANT USAGE ON DATABASE PROD_DB TO ROLE AVA_ROLE;
GRANT USAGE ON ALL SCHEMAS IN DATABASE PROD_DB TO ROLE AVA_ROLE;
GRANT SELECT ON ALL TABLES IN DATABASE PROD_DB TO ROLE AVA_ROLE;
GRANT SELECT ON ALL VIEWS IN DATABASE PROD_DB TO ROLE AVA_ROLE;
GRANT USAGE ON ALL FUNCTIONS IN DATABASE PROD_DB TO ROLE AVA_ROLE;

-- For metadata access
GRANT IMPORTED PRIVILEGES ON DATABASE SNOWFLAKE TO ROLE AVA_ROLE;

-- Assign role to user
GRANT ROLE AVA_ROLE TO USER ava_service;
```

## Databricks Integration

Connect to Databricks lakehouse platform:

```bash
ava connect databricks \
  --name "Production Databricks" \
  --workspace-url "https://your-workspace.cloud.databricks.com" \
  --token "YOUR_ACCESS_TOKEN" \
  --catalog "main" \
  --schema-pattern "*"
```

### Generate Databricks Access Token

<Steps>
  <Step title="Open User Settings">
    Click your profile icon > **User Settings**
  </Step>
  <Step title="Generate Token">
    Navigate to **Access Tokens** > **Generate New Token**
  </Step>
  <Step title="Configure Token">
    - **Comment**: AVA Integration
    - **Lifetime**: 90 days (or as per policy)
  </Step>
  <Step title="Copy Token">
    Save the token securely - it's shown only once
  </Step>
</Steps>

## Cloud Storage

### Amazon S3

```bash
ava connect s3 \
  --name "Production S3" \
  --access-key-id "YOUR_ACCESS_KEY" \
  --secret-access-key "YOUR_SECRET_KEY" \
  --region "us-east-1" \
  --buckets "prod-data,customer-uploads"
```

```bash

  --name "Production S3" \

  --region "us-east-1" \
  --buckets "prod-data,customer-uploads"
```

## Custom Connectors

Create custom connectors for proprietary systems:

<Tabs>
  <Tab title="REST API">
    ```python
    from ava.connectors import CustomConnector

    class MyAPIConnector(CustomConnector):
        def __init__(self, config):
            super().__init__(config)
            self.api_url = config['api_url']
            self.api_key = config['api_key']

        def discover(self):
            # Implement discovery logic
            response = requests.get(
                f"{self.api_url}/tables",
                headers={"Authorization": f"Bearer {self.api_key}"}
            )
            return self.parse_tables(response.json())

        def scan_table(self, table_name):
            # Implement table scanning
            response = requests.get(
                f"{self.api_url}/tables/{table_name}/data",
                headers={"Authorization": f"Bearer {self.api_key}"}
            )
            return self.parse_data(response.json())

    # Register connector
    client.connectors.register('my-api', MyAPIConnector)

    # Use connector
    client.connect('my-api', {
        'name': 'My Custom API',
        'api_url': 'https://api.mycompany.com',
        'api_key': 'YOUR_API_KEY'
    })
    ```
  </Tab>

  <Tab title="JDBC">
    ```bash
    ava connect jdbc \

      --url "jdbc:mysql://legacy.company.com:3306/production" \

      --driver-path /path/to/mysql-connector.jar
    ```

## Connection Management

### List Connections

```bash
# List all connections
ava connections list

# Show connection details
ava connections show salesforce

# Test connection
ava connections test salesforce
```

### Update Connection

```bash
# Update credentials
ava connections update salesforce \
  --client-secret "NEW_SECRET"

# Update configuration
ava connections configure salesforce \
  --scan-frequency "daily" \
  --include-sandboxes true
```

### Delete Connection

```bash
ava connections delete salesforce --confirm
```

<Warning>
  Deleting a connection will remove all discovered metadata. Ensure you have backups before proceeding.
</Warning>

## Data Discovery Configuration

### Scan Profiles

Choose the appropriate scan profile for each source:

<Tabs>
  <Tab title="Quick Scan">
    ```bash
    ava scan start --source salesforce --profile quick
    ```
    - Scans metadata only
    - Fast (minutes)
    - No data sampling
    - Good for initial discovery
  </Tab>

  <Tab title="Standard Scan">
    ```bash
    ava scan start --source salesforce --profile standard
    ```
    - Scans metadata + schema
    - Moderate speed (hours)
    - Sample data analysis
    - Recommended for most use cases
  </Tab>

  <Tab title="Deep Scan">
    ```bash
    ava scan start --source salesforce --profile deep
    ```
    - Full metadata + data profiling
    - Slow (hours to days)
    - Complete data analysis
    - Best for compliance requirements
  </Tab>
</Tabs>

### Schedule Scans

```bash
# Schedule daily scans
ava scan schedule --source salesforce \
  --profile standard \
  --frequency daily \
  --time "02:00"

# Schedule weekly deep scans
ava scan schedule --source snowflake \
  --profile deep \
  --frequency weekly \
  --day sunday \
  --time "01:00"
```

### Connection Health

```bash
# Check all connection health
ava health check --type connections

ava connections monitor salesforce --watch

# View connection metrics
ava connections metrics salesforce
```

### Troubleshooting

<AccordionGroup>
  <Accordion title="Connection timeout">
    **Solutions:**
    1. Check network connectivity
    2. Verify firewall rules
    3. Increase timeout setting
    ```bash
    ava connections configure salesforce --timeout 60
    ```
  </Accordion>

  <Accordion title="Authentication failed">
    **Check:**
    1. Credentials are correct
    2. Account hasn't expired
    3. Permissions are sufficient
    4. IP whitelisting is configured
    ```bash
    ava connections test-auth salesforce --verbose
    ```
  </Accordion>

  <Accordion title="Partial data discovery">
    **Verify:**
    1. Service account has all required permissions
    2. No rate limiting is occurring
    3. All schemas/databases are accessible
    ```bash
    ava scan logs --source salesforce --last-scan
    ```
  </Accordion>
</AccordionGroup>

## Security Best Practices

<CardGroup cols={2}>
  <Card title="Use Service Accounts" icon="user-gear">
    Create dedicated service accounts with minimal required permissions
  </Card>
  <Card title="Rotate Credentials" icon="arrows-rotate">
    Regularly rotate passwords and API keys (every 90 days)
  </Card>
  <Card title="Use Key-Based Auth" icon="key">
    Prefer key pairs or OAuth over password authentication
  </Card>
  <Card title="Encrypt Credentials" icon="lock">
    AVA encrypts all stored credentials at rest
  </Card>
  <Card title="Network Isolation" icon="network-wired">
    Use private networking or VPNs for database connections
  </Card>
  <Card title="Monitor Access" icon="eye">
    Enable audit logging for all connection activities
  </Card>
</CardGroup>
